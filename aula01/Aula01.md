1. O que é Git e por que usamos (controle de versão)

 O Git é uma ferramenta (software) que instalamos no computador. Ele serve para monitorar o histórico de alterações de qualquer arquivo

2. O que é GitHub (onde guardamos e compartilhamos): 

GitHub é como uma 'rede social' de programadores. Serve como porta de entrada para conseguir entrar no mercado de trabalho. Considerado o  portifólio de um desenvolvedor, é onde postamos os projetos de autoria própria.

3. O que é um repositório:

Repositório é a onde armazenamos os projetos que subimos para a plataforma GitHub.

4. O que são commit, push, pull, branch (definições curtas + 1 analogia):

Commit: Salva as alterações feitas no projeto na máquina atual. É o 'checkpoint' do projeto

 Push: Comando responsável por enviar ao GitHub as alterações salvas pelo Commit. É como fazer 'upload' de arquivos para a plataforma.

 Pull: Traz as alterações do repositório para a máquina atual. É como passar o código de um computador para outro, A - B.

 Branch: Cria uma 'linha do tempo' paralela para testar coisas novas sem estragar o projeto atual. Serve como rascunho/cópia do projeto,em casos de fazer algo no projeto que não funcione, o 'Branch' salva tudo em uma linha do tempo.
